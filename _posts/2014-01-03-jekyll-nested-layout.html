First of all, this is the first post of my ever-dreamed personal blog. Congratulations to myself as this is a great step for my developer life!

Back to the topic, this site is powered by Github page which in fact uses Jekyll as the backend engine. I started this blog with little knowledge to Jekyll so I built from scratch and learnt the how-tos of Jekyll step by step.

One of the most difficult tasks was to layout. I had to do mix-and-match many different pieces of HTML, CSS and Javascripts to make the site possible. How good for me to place these code pieces in a proper location so that future maintenance can be easier? this is a big question to a Jekyll newbie...

When I came to work out a template of post, I didn't want to have another layout file as I had already a layout file to style other pages e.g. front and about pages. I was hestiated to start a new layout file as this will be duplicating my work on other pages.

I was stuck a while, thought to apply the exisiting layout (which is app.html) and include head and foot files to make a container to contain the post:

---
layout: app
---
{% include post_head.md %}
*the post content*
{% include post_foot.md %}

NO LUCK! I found that this is not permitted as the header include file contains a non-closing div tag.

I had no choice but to re-think another way.

From my few-years knowing of Ruby on Rails in which nested layout is possible, I came up with an idea to try it in Jekyll. I created another layout file called post.html, and its front-matter was:

_layout/post.html

---
layout: app
---
<div class="post_content">
  *the post content*
<div>

Tested it, it WORKED like a charm!
